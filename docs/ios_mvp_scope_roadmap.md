# iOS AI 投资顾问 1 个月 MVP 范围

版本：v0.1  
日期：2026-05-21  
关联 PRD：`docs/ios_ai_investment_advisor_prd.md`

## 1. MVP 目标

1 个月内交付一个可供 50-100 名种子用户试用的 iOS MVP，验证两个问题：

- 用户是否会反复使用“单股 AI 问诊”来辅助 A 股投资决策。
- “建议型 AI 投顾”是否能在免责声明、风险提示、证据链和输出边界上做到合规可控。

MVP 不追求功能完整，优先做出可信、可用、可测量的核心闭环。

## 2. MVP 核心闭环

用户打开 App -> 搜索 A 股 -> 查看实时行情/新闻/公告摘要 -> 发起 AI 问诊 -> 获得建议型结论和完整报告 -> 保存报告或加入持仓 -> 用完免费额度后看到订阅权益。

## 3. 必须做

### P0: 单股问诊闭环

- A 股代码/名称搜索。
- 股票详情基础信息：名称、代码、市场、最新价、涨跌幅、数据更新时间。
- AI 问诊入口。
- 输出买入、持有、减仓、卖出或观望倾向。
- 输出信心分数。
- 输出参考买入区间、止盈区间、止损区间。
- 输出短线、波段、中长期观点。
- 输出核心理由：技术面、基本面、消息面、资金面。
- 输出可展开的完整报告。
- 数据失败或不完整时，明确提示并降低结论确定性。

### P0: 实时数据最小接入

- 行情：至少支持最新价、涨跌幅、成交量或成交额、更新时间。
- 新闻：至少展示最近新闻标题、来源、时间。
- 公告：至少展示最近公告标题、来源、时间。
- 数据来源和更新时间必须可见。

说明：MVP 可以先接入一个可用数据源，先保证闭环稳定，不做多数据源融合。

### P0: 合规与风控

- 首次进入 App 展示风险提示和用户确认。
- 每份报告底部展示免责声明。
- 禁止承诺收益、保证准确、保本、稳赚、确定性荐股等表达。
- AI 输出必须包含置信度、理由和风险提示。
- 后端记录每次问诊的输入、数据时间、输出、模型版本和用户确认状态，便于追踪。

### P0: 免费额度与订阅权益展示

- 免费用户有明确问诊次数限制。
- 用户可看到剩余额度。
- 额度耗尽后展示订阅权益页。
- 订阅权益页展示 Free / Pro / Premium 三档差异。
- 记录订阅页曝光、点击和用户反馈，用于验证付费意愿。

说明：如果 Apple IAP 配置、审核或开发排期有风险，MVP 可以先做“订阅权益展示 + 付费意愿收集”，把真实扣费放到 TestFlight 后续版本。

### P0: 种子用户验证基础能力

- 基础账号或设备级用户识别。
- 事件埋点：搜索、问诊发起、问诊完成、完整报告展开、报告保存、额度耗尽、订阅页曝光。
- 简单反馈入口：本次建议是否有用、是否可信、是否愿意付费。

## 4. 应该做但可降级

### P1: 持仓体检轻量版

必须保留为辅助路径，但范围要压缩：

- 用户手动添加持仓股票。
- 可选录入成本价和仓位。
- 展示持仓列表。
- 对单只持仓复用“单股问诊”。
- 输出组合层面的简单风险摘要。

暂不做复杂组合优化、自动调仓和券商同步。

### P1: 历史报告轻量版

- 保存最近若干份问诊报告。
- 支持历史报告列表和详情查看。
- 免费版限制保存数量，付费档位展示更高保存权益。

如果时间紧，历史报告只做本地或后端简单存储，不做复杂搜索、筛选和对比。

### P1: 订阅档位配置

建议 MVP 先定义权益，不急于精确定价：

| 档位 | MVP 权益 |
|---|---|
| Free | 每日少量单股问诊、简版结论、有限历史保存 |
| Pro | 更多问诊次数、完整报告、更强模型、更多历史保存 |
| Premium | 更高额度、持仓体检、完整多智能体流程、优先分析资源 |

## 5. 砍掉

以下内容不进 1 个月 MVP：

- 真实交易下单。
- 券商账户绑定和自动同步持仓。
- 价格、公告、新闻推送提醒。
- 交易复盘。
- 港股、美股、ETF、基金完整支持。
- 多数据源质量仲裁。
- 复杂量化回测。
- 社区、排行榜、跟单、组合公开分享。
- PDF/图片报告导出。
- iPad、macOS、Android 适配。
- 完整客服、工单、运营后台。
- 自动续订订阅的复杂促销、优惠码、家庭共享等高级能力。

砍掉原则：凡是不能直接验证“用户是否需要单股 AI 问诊”和“建议型输出是否合规可控”的功能，全部后置。

## 6. 每周交付计划

### 第 1 周：产品骨架和技术闭环

目标：跑通从 iOS 到后端到 AI 问诊的最小链路。

交付物：

- iOS App 基础工程和导航结构。
- 首次风险提示页。
- 股票搜索页原型。
- 股票详情页基础布局。
- 后端新增或整理移动端 API 草案。
- 单股问诊 API 最小版本。
- 一只或少量测试股票的行情、新闻、公告数据接入。
- AI 输出 JSON 结构定义。
- 合规敏感词和禁止表达清单 v0.1。

验收标准：

- 用户可以在 iPhone 模拟器中搜索股票并进入详情。
- 可以发起一次问诊并看到结构化结果。
- 问诊结果包含结论、信心分数、操作区间和核心理由。

### 第 2 周：单股问诊体验成型

目标：让主路径达到可用，而不是只跑通。

交付物：

- 股票搜索体验完善：搜索、空状态、错误状态。
- 股票详情展示真实行情更新时间。
- 新闻和公告摘要展示。
- 单股问诊结果页完整信息架构。
- 完整报告展开/收起。
- 数据失败降级提示。
- 报告底部免责声明。
- 问诊次数扣减逻辑。
- 基础事件埋点。

验收标准：

- 主路径可以连续完成：搜索 -> 详情 -> 问诊 -> 查看报告。
- 数据源异常时不会输出误导性强结论。
- 用户能清楚看到结论依据和风险提示。

### 第 3 周：商业化和辅助路径

目标：补齐免费/付费验证和轻量持仓体检。

交付物：

- Free / Pro / Premium 权益页。
- 免费额度展示和额度耗尽状态。
- 订阅页曝光和点击埋点。
- 付费意愿反馈入口。
- 手动添加持仓。
- 持仓列表。
- 对持仓股票发起问诊。
- 简单组合风险摘要。
- 历史报告轻量保存。

验收标准：

- 免费用户可以看到剩余额度，并在额度耗尽后进入订阅权益页。
- 用户可以添加持仓，并对持仓股票发起问诊。
- 系统能保存和查看最近报告。

### 第 4 周：内测、合规修正和 TestFlight 准备

目标：把 MVP 从“能用”打磨到“可给种子用户试用”。

交付物：

- 10-20 名内部/种子用户试用。
- 收集建议有用性、可信度、付费意愿反馈。
- 修正高风险文案和误导性输出。
- 完善错误状态、加载状态、超时状态。
- 问诊日志和基础审计记录检查。
- 性能优化：问诊等待体验、超时处理。
- TestFlight 包或等价内测包。
- MVP 复盘看板：留存、问诊完成率、报告展开率、订阅页访问率。

验收标准：

- 主要路径无阻塞性崩溃。
- 单股问诊完成率可被统计。
- 合规风险文案经过至少一轮人工审查。
- 可邀请 50-100 名种子用户试用。

## 7. 发布口径

MVP 对外建议避免直接使用强监管暗示的“AI 投资顾问”作为首屏主标题，优先使用更稳妥表达：

- AI 投资分析助手
- AI 股票分析助手
- A 股智能分析助手

产品内部可以继续以“建议型 AI 投顾”作为工作定义，但用户侧文案应强调“分析参考，不构成投资建议或收益承诺”。

## 8. MVP 成功标准

必须达到：

- 50-100 名种子用户完成试用邀请。
- 单股问诊完成率达到 60% 以上。
- 建议有用/可信反馈达到 70% 以上。
- 0 个严重合规或误导性表达问题进入外部测试版本。

理想达到：

- 7 日留存达到 20%-30%。
- 完整报告展开率达到 40% 以上。
- 免费额度耗尽用户中，30% 以上进入订阅权益页。
- 10% 以上试用用户表达明确付费意愿。

## 9. 决策点

第 2 周末必须决策：

- 数据源是否足够稳定支撑种子用户测试。
- 单股问诊平均耗时是否可接受。
- AI 输出是否需要进一步收紧建议强度。

第 3 周末必须决策：

- 真实 Apple IAP 是否进入 MVP，还是只做订阅权益验证。
- 持仓体检是否保留在第一轮种子用户测试。
- Free / Pro / Premium 的初始额度是否需要调整。

第 4 周末必须决策：

- 是否扩大 TestFlight 用户规模。
- 是否进入真实订阅开发和 App Store 审核准备。
- 是否把下一阶段重点放在提醒、持仓增强或付费转化。
