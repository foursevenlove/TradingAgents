# iOS AI 股票分析助手 MVP 决策清单

版本：v0.1  
日期：2026-05-21  
用途：MVP 启动前/第 1 周评审/第 4 周是否扩大测试的拍板清单  

## 1. 决策摘要

默认建议：先做“AI 股票分析助手”的 1 个月封闭内测 MVP，不公开以“AI 投资顾问”收费上线。

MVP 只验证两件事：

- 用户是否真的需要单股 AI 问诊。
- 建议型输出是否能在合规、数据来源、解释链和成本上可控。

## 3. 默认建议详表

### 3.1 产品定位

最终决策：

- 对外是否使用“AI 投资顾问”。

默认建议：

- 不使用。
- 对外使用“AI 股票分析助手”或“A 股智能分析助手”。
- 内部可以继续把它称作“建议型 AI 投顾”作为工作定义。

理由：

- “AI 投资顾问 + 买卖倾向 + 订阅收费”叠加后，合规风险最高。
- MVP 目标是验证需求和输出边界，不是第一天挑战监管边界。

拍板条件：

- 如果没有专业法律/合规意见支持，不进入公开收费。

### 3.2 目标市场

最终决策：

- 第一版只做 A 股，还是同时做港股/美股/ETF。

默认建议：

- 第一版只做 A 股。
- 港股、美股、ETF 全部后置。

理由：

- 当前 PRD 的核心用户是个人投资者和 A 股活跃交易者。
- 多市场会显著增加数据授权、交易规则和报告质量复杂度。

### 3.3 MVP 主路径

最终决策：

- 单股问诊、持仓体检、每日顾问、事件提醒、交易复盘谁优先。

默认建议：

- P0：单股问诊。
- P1：持仓体检轻量版。
- P2：提醒、复盘、多市场。

理由：

- 单股问诊最短路径验证用户是否愿意主动使用。
- 持仓体检可以复用单股问诊，不应抢主路径资源。

### 3.4 AI 输出边界

最终决策：

- 是否输出买入、持有、减仓、卖出。

默认建议：

- UI 展示“分析倾向”：偏积极、观望、偏谨慎、规避。
- 内部可以映射到买入/持有/减仓/卖出，但用户侧避免命令式表达。

必须项：

- 信心分数。
- 短中长期观点。
- 核心理由。
- 主要风险。
- 反方观点。
- 结论失效条件。
- 数据来源和更新时间。
- 免责声明。

默认不做：

- “建议你买入/卖出”。
- “今日必买”。
- “精准买卖点”。
- “收益承诺”。

### 3.5 操作区间

最终决策：

- MVP 是否展示买入区间、止盈、止损。

默认建议：

- 先弱化为“参考区间/风险区间”。
- 如果程序化规则无法校验，就不展示具体价格。
- 对 Free 默认不展示完整操作区间。

理由：

- 操作区间是高风险输出，容易被理解为明确投顾建议。
- 数据不完整或行情延迟时，具体价位误导风险较高。

### 3.6 数据源

最终决策：

- 用什么数据源支撑 MVP 和商业上线。

默认建议：

- 研发/封闭内测：可以暂用 AkShare/Tushare。
- 公开收费：必须换成或补充可商用授权数据源。

必须确认：

- 行情是否可商用。
- 新闻是否可展示、缓存、加工。
- 公告是否可展示、缓存、加工。
- 是否允许 AI 分析加工后对订阅用户提供。

闸门：

- 没有可商用授权数据源，不公开收费。

### 3.7 订阅支付

最终决策：

- MVP 是否接真实 Apple IAP。

默认建议：

- P0 不做真实扣费。
- P0 做订阅权益页、额度控制、付费意愿收集。
- P1 再接 StoreKit 2 和后端 transaction 校验。

理由：

- 合规和数据授权没确认前，真实收费会放大风险。
- 先验证权益是否有吸引力，再投入完整支付链路。

### 3.8 定价

最终决策：

- Pro / Premium 初始价格。

默认建议：

- Pro：¥38/月，¥298/年。
- Premium：¥98/月，¥798/年。

备选低价测试：

- Pro：¥28/月，¥198/年。
- Premium：¥78/月，¥598/年。

默认选择：

- 对外正式价用 ¥38 / ¥98。
- 种子用户通过兑换码或限时折扣降低门槛，不把正式价定太低。

### 3.9 权益限制

最终决策：

- Free / Pro / Premium 各自额度。

默认建议：

| 档位 | 轻量问诊 | 深度问诊 | 持仓体检 | 历史报告 |
|---|---:|---:|---:|---:|
| Free | 3 次/日 | 1 次/周 | 不开放 | 最近 3 份 |
| Pro | 20 次/日 | 3 次/日 | 轻量持仓查看 | 200 份或 180 天 |
| Premium | 50 次/日 | 8 次/日 | 1 次/日 | 1000 份或 2 年 |

必须限制：

- 不允许“无限深度分析”。
- 深度报告和持仓体检必须限频。

### 3.10 模型和成本

最终决策：

- 哪些模型用于 Free / Pro / Premium。

默认建议：

- Free Light：qwen-turbo 非思考。
- Pro Deep：qwen-plus 思考或 deepseek-v4-flash。
- Premium Deep/Portfolio：qwen-plus 思考。
- GPT-5 mini 只作为质量对照或海外备选，不作为国内 MVP 默认路径。

第 2 周必须确认：

- 单次 Light token、耗时、成本。
- 单次 Deep token、耗时、成本。
- Premium 高频用户满额成本是否可接受。

成本闸门：

- 如果满额使用下 Pro/Premium 模型成本过高，必须降低深度额度或降级模型。

### 3.11 技术范围

最终决策：

- 是否按五模块同步推进。

默认建议：

- iOS：SwiftUI，先完成主路径。
- 后端：新增 `/api/mobile/*`，复用现有 FastAPI、任务、持仓、token usage。
- AI：先 Light，后 Deep。
- 数据源：snapshot 聚合 + 数据新鲜度 + 数据快照。
- 订阅：先 entitlement/quota，再 StoreKit。

MVP 不做：

- 真实交易下单。
- 券商绑定。
- 推送提醒。
- 自动调仓。
- 社区和跟单。
- PDF/图片导出。

### 3.12 用户验证

最终决策：

- 第 1 个月用什么指标判断继续投入。

默认建议：

必须达成：

- 50-100 名种子用户完成试用邀请。
- 单股问诊完成率 >= 60%。
- 建议有用/可信反馈 >= 70%。
- 严重合规或误导性输出 = 0。

理想达成：

- 7 日留存 20%-30%。
- 完整报告展开率 >= 40%。
- 免费额度耗尽用户中，30% 以上进入订阅权益页。
- 10% 以上试用用户表达明确付费意愿。

## 4. Go / No-Go 闸门

### 进入开发前

必须 Go：

- 产品名称和发布口径确认。
- MVP 只做封闭内测还是公开发布确认。
- 数据源授权状态标记。
- 风险提示和免责声明 v0.1。
- Free / Pro / Premium 权益初版。

No-Go：

- 坚持公开使用“AI 投资顾问”并收费，但没有合规结论。
- 无法接受输出降级为“分析倾向”。
- 无法接受真实 IAP 后置。

### 第 2 周末

必须 Go：

- 单股问诊主路径可用。
- 数据新鲜度和降级策略可用。
- token usage 和单次成本可统计。
- 结构化报告解析稳定。

No-Go：

- 数据经常缺失且仍输出强结论。
- 单次问诊成本或耗时明显失控。
- 模型无法稳定输出可校验 JSON。

### 第 4 周末

必须 Go：

- 合规文案经过人工审查。
- 20-30 只股票评测集通过。
- 种子用户反馈显示“有用/可信”达到目标。
- 所有报告可追溯数据快照和模型 usage。

No-Go：

- 出现严重误导性输出。
- 无法追溯报告依据。
- 数据授权无法支撑下一阶段公开测试。
- 用户反馈认为报告不可信或不可行动。

## 5. 当前默认决策包

如果今天必须拍板，建议采用以下组合：

- 产品名：AI 股票分析助手。
- 发布方式：封闭 TestFlight / 种子用户内测。
- MVP 主路径：单股问诊。
- 辅助路径：轻量持仓体检。
- 输出边界：分析倾向，不做强买卖指令。
- 操作区间：默认弱化，无法校验则不展示。
- 数据源：AkShare/Tushare 仅内测，商业源另行确认。
- 订阅：P0 只做权益和额度，真实 IAP 后置。
- 定价假设：Pro ¥38/月，Premium ¥98/月。
- Free 额度：每日 3 次轻量问诊。
- 模型路由：Free 轻量，Pro/Premium 深度限频。
- 成功标准：有用/可信 >= 70%，严重合规问题 = 0。

## 6. 仍需外部确认

- 是否有证券投资咨询相关资质或合作主体。
- 可商用 A 股行情、公告、新闻数据源。
- iOS 开发者账号和 App Store Connect 权限。
- 是否要在 MVP 使用手机号/Apple 登录，还是匿名设备 ID。
- LLM API 预算上限。
- 是否需要专业律师出具上线前合规意见。
